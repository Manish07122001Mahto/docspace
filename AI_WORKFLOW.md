# AI Workflow Note

## AI Tools Used

The following AI tools were used during development:

* ChatGPT
* GitHub Copilot

## How AI Helped

AI was primarily used to accelerate implementation rather than make engineering decisions.

Examples include:

* Generating initial boilerplate code
* Explaining unfamiliar APIs
* Drafting Prisma schema ideas
* Suggesting React component structures
* Reviewing implementation approaches
* Improving documentation wording

## What I Changed or Rejected

AI-generated code was reviewed before being incorporated into the project.

Several generated suggestions were simplified or rewritten to better fit the project requirements.

Examples include:

* Avoiding unnecessary abstraction layers
* Removing overly complex authentication approaches
* Simplifying API responses
* Adjusting database models for the required scope
* Refactoring generated code for readability and consistency

## Verification Process

Every AI-assisted change was manually verified before being accepted.

Verification included:

* Manual end-to-end testing
* API testing
* Reviewing database changes
* Validating document persistence
* Confirming sharing behavior
* Running automated tests

## Development Philosophy

AI was treated as a productivity tool rather than a replacement for engineering judgment.

Final architectural decisions, implementation details, debugging, testing, and trade-off decisions were made manually to ensure the application remained focused, maintainable, and aligned with the assignment requirements.
